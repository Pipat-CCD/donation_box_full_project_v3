# ระบบเก็บกล่องบริจาคผ่านมือถือ (Donation Box App)

เวอร์ชันนี้ปรับตามรายการแก้ไขล่าสุดแล้ว พร้อมอัปโหลดขึ้น GitHub และ Streamlit Community Cloud ได้ทันที

## สิ่งที่เพิ่ม/แก้ไขแล้ว

1. เมื่อบันทึกสำเร็จ มีข้อความแจ้งเตือนสีเขียวชัดเจนว่า **บันทึกเรียบร้อยแล้ว**
2. รองรับการเก็บข้อมูลบน Google Sheets และสำรอง CSV/รูปภาพเข้า Google Drive
3. GPS ใช้แบบ **วางลิงก์ Google Maps** แทนการบันทึกพิกัดโดยตรง
4. เพิ่ม / ลบ / แก้ไข รายการกล่องได้
5. มีแพทเทิร์น CSV สำหรับอัปโหลดรายการกล่องทั้งหมดหรือบางรายการ
6. การบันทึกการเก็บ **ไม่ต้องใส่จำนวนเงิน**
7. มีระบบสร้างบัญชีผู้ใช้ เพื่อดูว่าเจ้าหน้าที่คนใดเป็นผู้บันทึก
8. ดาวน์โหลดข้อมูลได้ทุกตาราง
9. ตัวหนังสือใหญ่ เด่นชัด
10. หน้าเว็บสีสันอ่านง่าย เหมาะกับผู้สูงอายุ
11. หน้าแผนที่/นำทาง แสดงรูปที่บันทึกไว้ของแต่ละกล่อง

## บัญชีเริ่มต้น

```text
ชื่อผู้ใช้: admin
รหัสผ่าน: 1234
```

ควรเปลี่ยนรหัสผ่านทันทีเมื่อใช้งานจริง หรือสร้างบัญชีเจ้าหน้าที่ใหม่ในเมนู **บัญชีผู้ใช้**

## การอัปโหลดรายการกล่อง

เข้าเมนู **อัปโหลดรายการกล่อง** แล้วดาวน์โหลดแพทเทิร์น CSV

คอลัมน์ที่ต้องมี:

```text
box_id, store_name, province_zone, subzone, contact, landmark, google_maps_url, cycle_months, status, last_collect_date, next_due_date, note
```

เลือกได้ 2 แบบ:

- **เพิ่ม/อัปเดตบางรายการ**: ถ้าเลขกล่องซ้ำ จะอัปเดตข้อมูลเดิม ถ้าเลขกล่องใหม่ จะเพิ่มรายการใหม่
- **แทนที่รายการทั้งหมด**: ล้างรายการเดิม แล้วใช้ไฟล์ที่อัปโหลดแทนทั้งหมด

## การเชื่อม Google Drive / Google Sheets

ระบบทำงานได้ 2 โหมด

1. CSV Local: ใช้งานได้ทันที ไม่ต้องตั้งค่า Google
2. Google Sheets + Drive: ใช้งานจริงหลายคน ข้อมูลอยู่บน Google Drive/Google Sheets

ให้ดูขั้นตอนในไฟล์:

```text
docs/GOOGLE_SETUP.md
```

## การ Deploy

1. อัปโหลดไฟล์ทั้งหมดในโฟลเดอร์นี้ขึ้น GitHub
2. เข้า Streamlit Community Cloud
3. เลือก Repository
4. Main file path ใส่ `streamlit_app.py`
5. กด Deploy

## อัปเดตล่าสุด: หน้าเพิ่ม/ลบ/แก้ไข

### หน้า “บัญชีผู้ใช้ / เจ้าหน้าที่”
เพิ่มความสามารถครบ 4 ส่วน
1. เพิ่มบัญชีใหม่
2. แก้ไขบัญชีเดิม เช่น ชื่อผู้ใช้ รหัสผ่าน ชื่อที่แสดง สิทธิ์ และสถานะ
3. ปิดใช้งานหรือลบบัญชี
4. ดาวน์โหลดบัญชีผู้ใช้แบบไม่แสดงรหัสผ่าน

> แนะนำให้ “ปิดใช้งาน” มากกว่า “ลบถาวร” เพื่อเก็บประวัติว่าเจ้าหน้าที่คนใดเคยบันทึกข้อมูล

### หน้าที่ควรมี เพิ่ม/ลบ/แก้ไข
- บัญชีผู้ใช้: เพิ่ม/แก้ไข/ปิดใช้งาน/ลบ เฉพาะผู้ดูแลระบบ
- รายการกล่อง: เพิ่ม/แก้ไข/ลบกล่อง เฉพาะผู้ดูแลระบบ
- แจ้งกล่องหาย: ควรแก้ไขสถานะเคสได้ เช่น รอตรวจสอบ/ปิดเคส/ต้องวางใหม่
- ประวัติการเก็บ: ควรแก้ไขหรือลบบันทึกผิดพลาดได้ แต่จำกัดเฉพาะผู้ดูแลระบบ
- อัปโหลดรายการกล่อง: ควรใช้เพิ่ม/อัปเดตแบบหลายรายการ และดาวน์โหลดแพทเทิร์นก่อนนำเข้า

## ปรับปรุงเวอร์ชัน v5

### เพิ่มหน้า “วางกล่องใหม่” ให้ User
- ผู้ใช้สิทธิ์ `user` จะเห็นเมนู **วางกล่องใหม่** เพิ่มในหน้าใช้งานมือถือ
- User เพิ่มจุดวางใหม่ได้ แต่ระบบจะบันทึกสถานะเป็น **รออนุมัติ**
- User ไม่สามารถลบหรือแก้ไขรายการหลักหลังบันทึก เพื่อป้องกันข้อมูลสูญหาย
- User ดูรายการที่ตนเองส่งคำขอและยังรออนุมัติได้

### เพิ่มระบบอนุมัติสำหรับ Admin
หน้า **วางกล่องใหม่ / จัดการกล่อง** ของ Admin เพิ่มแท็บ
1. วางกล่องใหม่
2. อนุมัติคำขอ
3. แก้ไขข้อมูลกล่อง
4. ลบกล่อง

Admin สามารถอนุมัติคำขอ User ให้เปลี่ยนสถานะจาก **รออนุมัติ** เป็น **ใช้งาน** หรือส่งกลับเป็น **รอแก้ไข** ได้
